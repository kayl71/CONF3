<h1>Ассемблер и интерпретатор</h1>

<h2>Цель</h2>
Разработать ассемблер и интерпретатор для учебной виртуальной машины (УВМ).

<h2>Описание проекта</h2>
<h3>Сборка и запуск</h3>
Для запуска приложения требуется установить все файлы репозитория, после чего запустить программу через главный скрипт main.py:

<code>python3 main.py</code>

<h3>Принцип работы</h3>
В файл json (input.json) в теле <code>"program"</code> пишется ассемблерный код, который представляет собой массив списков из 3 значений : [command, val1, val2], где command - название команды, val1 - первый аргумент, val2 - второй аргумент. Затем ассемблерный код транслируется в машинное представление в бинарный файл (output.txt). После программа считывает бинарный файл и выполняет все команды с выводом описания работы каждой. В конце выводится значение памяти в десятичной и бинарной форме.

Команды ассемблера
<ol>
  <li> <code>"LOAD_CON"</code> - загрузка константы. val1 - Значение константы; val2 - адрес в памяти, где будет хранится константа </li>
  <li> <code>"READ"</code> - чтение из памяти. val1 - адрес, где хранится значение </li>
  <li> <code>"WRITE"</code> - запись в память. val1 - адрес, откуда берется значение; val2 - адрес, куда записывается значение </li>
  <li> <code>"POW"</code> - возведение в степень. val1 - адрес, где хранится значение основания операции и куда записывается результат; val2 - адрес, где хранится значение показателя операции </li>
</ol>

Размер памяти size_bytes можно менять, по умолчанию 64 байта. Размер значения value_size_bytes можно менять, по умолчанию 4 байта. Количество допустимых адресов равно <code> (size_bytes-value_size_bytes)*8 </code>. 
При записи значения в память, сначало обнуляются кол-во байт, равному значению value_size_bytes, начиная с номера байта, которое и является адресом. Затем записывается значение. Таким образом, если например value_size_bytes = 4, то при записанном значении x в адрес 10, можно счтитать как с адреса 10 и получить x, так и с адреса 9 или 11 и получить x<<1 или x>>1 соответственно.
